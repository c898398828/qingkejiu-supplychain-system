# 互助县青稞酒产业链协同管理系统毕业设计文档

## 1. 引言

### 1.1 项目背景

互助县是中国最大的青稞酒生产基地，青稞酒产业是互助县的支柱产业。随着青稞酒产业的不断发展，产业链各环节之间的协同管理变得越来越重要。传统的管理方式已经无法满足现代产业发展的需求，需要一个现代化的信息系统来整合产业链资源，提高管理效率。

### 1.2 项目目标

本项目旨在开发一个互助县青稞酒产业链协同管理系统，实现从青稞种植、原料采购、生产加工、销售配送等全产业链的协同管理，提高产业整体效率和竞争力。

### 1.3 项目意义

- 提高产业链各环节的信息共享和协同效率
- 实现青稞酒质量的全程追溯
- 优化供应链管理，降低运营成本
- 为决策提供数据支持
- 促进青稞酒产业的可持续发展

## 2. 需求分析

### 2.1 功能需求

#### 2.1.1 系统管理
- 用户管理：用户注册、登录、权限管理
- 角色管理：角色创建、权限分配
- 权限管理：菜单权限、操作权限设置

#### 2.1.2 供应链管理
- 青稞种植管理：种植基地信息管理、种植计划管理
- 原料采购管理：采购计划、供应商管理、采购订单管理
- 库存管理：原料库存、成品库存、库存预警

#### 2.1.3 生产管理
- 生产流程管理：生产计划、生产任务、生产进度跟踪
- 质量管理：质量检测、质量记录、质量追溯

#### 2.1.4 销售管理
- 订单管理：订单创建、订单状态跟踪、订单统计
- 配送管理：配送计划、配送跟踪
- 客户管理：客户信息管理、客户分类

#### 2.1.5 协同办公
- 任务管理：任务分配、任务跟踪
- 文档管理：文档上传、文档共享
- 通知公告：系统通知、企业公告

#### 2.1.6 质量追溯
- 产品追溯：通过二维码或条形码追溯产品信息
- 追溯记录：记录产品从原料到成品的全过程信息

#### 2.1.7 数据分析
- 生产分析：生产效率、生产成本分析
- 销售分析：销售趋势、销售区域分析
- 供应链分析：供应链效率、库存周转率分析

### 2.2 非功能需求

- 系统响应时间：页面加载时间不超过3秒
- 系统稳定性：系统可用性达到99.9%
- 系统安全性：数据加密、权限控制
- 系统可扩展性：支持后续功能扩展
- 系统易用性：界面友好、操作简单

## 3. 系统设计

### 3.1 系统架构

本系统采用前后端分离的架构设计：

- 前端：Vue 3 + Vite + Element Plus
- 后端：Spring Boot + MyBatis-Plus + MySQL
- 数据库：MySQL 8.0

### 3.2 技术选型

| 分类 | 技术 | 版本 | 选型理由 |
| --- | --- | --- | --- |
| 前端框架 | Vue 3 | 3.4.21 | 轻量级、高性能、组件化开发 |
| 构建工具 | Vite | 5.2.0 | 快速的开发服务器和构建工具 |
| UI库 | Element Plus | 2.6.1 | 美观、易用、功能丰富的UI组件库 |
| 后端框架 | Spring Boot | 2.7.15 | 快速开发、内置Tomcat、自动配置 |
| ORM框架 | MyBatis-Plus | 3.5.3.1 | 简化MyBatis操作、提供代码生成 |
| 数据库 | MySQL | 8.0 | 稳定、可靠、广泛使用的关系型数据库 |

#### 4.1.2 核心页面实现

**登录页面**
- 实现用户登录功能，包括用户名、密码输入和登录验证
- 使用Element Plus表单组件，实现表单验证
- 登录成功后跳转到仪表盘页面

**仪表盘页面**
- 展示系统核心数据统计，包括种植面积、产量、销售金额、客户数量等
- 使用图表展示销售趋势和产品销售占比

**青稞种植管理页面**
- 实现种植基地信息的增删改查
- 支持按基地名称和状态进行搜索
- 分页展示种植基地列表

**原料采购管理页面**
- 实现采购记录的增删改查
- 支持按采购编号、供应商、原料名称和状态进行搜索
- 分页展示采购记录列表

**库存管理页面**
- 实现库存信息的增删改查
- 支持按产品名称、产品编号、类别和状态进行搜索
- 分页展示库存列表

1. 执行 `npm run build` 命令构建前端项目
2. 将构建后的 `dist` 目录部署到Web服务器（如Nginx、Apache）
3. 配置Web服务器的访问路径

### 6.2 后端部署

1. 执行 `mvn clean package` 命令构建后端项目
2. 将生成的 `jar` 文件部署到服务器
3. 执行 `java -jar barley-wine-system-1.0-SNAPSHOT.jar` 命令启动后端服务

### 6.3 数据库部署

1. 创建数据库 `barley_wine_system`
2. 执行 `db_init.sql` 脚本初始化数据库表结构和初始数据
3. 配置 `application.yml` 文件中的数据库连接信息

## 7. 结论与展望

### 7.1 项目总结

本项目成功实现了互助县青稞酒产业链协同管理系统，系统包括以下核心功能：

- 系统管理：用户、角色、权限管理
- 供应链管理：青稞种植、原料采购、库存管理
- 生产管理：生产流程、质量管理
- 销售管理：订单、配送、客户管理
- 协同办公：任务、文档、通知管理
- 质量追溯：产品追溯、追溯记录
- 数据分析：生产、销售、供应链分析

系统采用前后端分离的架构，前端使用Vue 3 + Vite + Element Plus，后端使用Spring Boot + MyBatis-Plus + MySQL，数据库使用MySQL 8.0。系统界面美观、操作简单、功能完整，能够满足互助县青稞酒产业链协同管理的需求。

## 8. 需求完善与系统重构（优化版）

### 8.1 优化后的核心需求

在原有需求基础上，补充以下可落地、可验收的新要求：

1. **统一业务接口规范**  
   所有模块统一为 `分页查询 + 新增 + 编辑 + 删除` 的 REST 风格接口，并统一返回结构 `{code, message, data}`。

2. **仪表盘经营视角增强**  
   除基础数量统计外，新增采购总额、销售总额、库存总值、种植总面积等经营指标，支撑毕业设计中的“数据驱动决策”目标。

3. **按模块配置驱动前端页面**  
   通过“模块配置表”统一定义字段、筛选项、展示列，减少重复页面开发成本，提高后期扩展效率。

4. **角色分工映射到系统模块**  
   将队友职责与业务模块绑定，在同一套系统中形成可协作开发边界，降低联调冲突。

5. **移动端兼容要求补充**  
   侧边栏与表格页面需在窄屏场景可用（导航折叠、表格横向滚动、弹窗可滚动）。

### 8.2 后端业务优化与新增要求

#### 8.2.1 后端结构优化

- 新增统一响应体 `ApiResponse` 与分页响应 `PageResponse`
- 新增 MyBatis-Plus 分页拦截器配置
- 新增 Spring Security 放行与 CORS 配置，保证前后端联调稳定

#### 8.2.2 新增与重构接口清单

- 认证接口：`POST /auth/login`
- 仪表盘接口：`GET /dashboard/overview`
- 系统管理：`/users`、`/roles`、`/permissions`
- 供应链：`/planting-bases`、`/purchases`、`/inventories`
- 生产管理：`/production-processes`、`/quality-controls`
- 销售管理：`/orders`、`/customers`

以上模块统一支持：
- `GET /xxx/page`（关键字+状态过滤+分页）
- `POST /xxx`（新增）
- `PUT /xxx/{id}`（编辑）
- `DELETE /xxx/{id}`（删除）

### 8.3 前端重构方案（基于当前UI风格）

保持现有“渐变背景 + 玻璃态卡片 + 左侧导航”视觉方向，重构为毕业设计业务信息架构：

1. 登录页：重定向到青稞酒系统业务入口  
2. 主布局：按“系统管理/供应链/生产/销售”分组导航  
3. 仪表盘：展示面积、采购、销售、库存等关键经营指标  
4. 通用模块页：统一承载各业务模块 CRUD，支持关键字、状态筛选、分页、弹窗表单
